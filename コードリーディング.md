## 2. 処理の流れ

```
[Frontend]
  InputForm → Run Agent クリック
      ↓
  App.tsx → POST /api/agent/run（→ Agent :3002 にプロキシ）

[Agent]
  routes/agent.ts
    ├── 1. Provider の /weather/price を fetch（価格取得）
    ├── 2. Provider の /weather/free を fetch（無料天気取得）
    ├── 3. services/aiDecision.ts → Claude APIで buy/skip 判断
    └── 4. buy の場合
          ├── @x402/fetch で Provider の /weather/premium にリクエスト
          ├── Provider の x402ミドルウェアが 402 を返す
          ├── @x402/fetch が自動でUSDC支払い
          └── 支払い確認後にプレミアム天気を返却

[Frontend]
  App.tsx → 結果をstateに保存
    ├── ResultDisplay → 結果表示
    └── LogDisplay    → ログ表示
```

### [Provider] `provider/src/config.ts` — 価格定数

```typescript
export const PREMIUM_PRICE = 0.01;
```

**価格はProviderが一元管理。** `index.ts`（paymentMiddleware）と `routes/weather.ts`（`/price` エンドポイント）の両方がここから参照する。  
Agentは価格をハードコードせず、Providerに問い合わせることで常に正しい価格を使う。

### [Provider] `provider/src/index.ts` — x402ミドルウェア設定

```typescript
import { paymentMiddleware, x402ResourceServer } from '@x402/express';
import { ExactEvmScheme } from '@x402/evm/exact/server';
import { HTTPFacilitatorClient } from '@x402/core/server';
```

**x402関連の3つのクラスをインポートしている。**

- `paymentMiddleware`：Expressのミドルウェア。指定したルートへのリクエストに対して支払いを要求する
- `x402ResourceServer`：支払いを受け付けるサーバーの設定
- `HTTPFacilitatorClient`：支払いの検証を外部のFacilitatorに委託するクライアント
- `ExactEvmScheme`：EVMチェーン（Base Sepolia）での支払いスキーム

```typescript
app.use(
  paymentMiddleware(
    {
      'GET /weather/premium': {       // ← 保護するルート
        accepts: [{
          scheme: 'exact',
          price: `$${PREMIUM_PRICE}`, // ← config.ts から参照
          network: 'eip155:84532',    // ← Base SepoliaのチェーンID
          payTo: EVM_ADDRESS,         // ← 受取アドレス
        }],
      },
    },
    new x402ResourceServer(facilitatorClient)
      .register('eip155:84532', new ExactEvmScheme()),
  ),
);
```

**ポイント：**
`/weather/premium` にリクエストが来ると、支払い証明があるかをチェックする。
なければ `402 Payment Required` を返し、あればFacilitatorで検証して通過させる。

### [Provider] `provider/src/services/mockWeather.ts` — 天気データ生成

```typescript
function getCondition(location: string): WeatherCondition {
  const index = (location.length * location.charCodeAt(0)) % conditions.length;
  return conditions[index];
}
```

**都市名の「文字数 × 先頭文字コード」を6で割った余りで天気を決定する。**

```
Tokyo: 5文字 × 84(T) = 420, 420 % 6 = 0 → 快晴
Paris: 5文字 × 80(P) = 400, 400 % 6 = 4 → 嵐
```

このアルゴリズムにより：
- 同じ都市名は常に同じ天気（デモの再現性）
- 外部APIに依存しない（ネット不要・高速）
- 都市名を変えるだけで全6種類の天気を試せる

### [Agent] `agent/src/routes/agent.ts` — エージェントのメインロジック

このファイルが一番重要。AIエージェントの「価格取得 → 判断 → 支払い → データ取得」を実装している。

```typescript
router.get('/price', async (_req, res) => {
  const r = await fetch(`${PROVIDER_URL}/weather/price`);
  res.json(await r.json());
});
```

**AgentはProviderから価格を取得してフロントエンドに返す。** Agentが価格を持たない設計。

```typescript
router.get('/balance', async (_req, res) => {
  const balance = await getUsdcBalance();
  res.json({ balance });
});
```

**ウォレットのUSDC残高をフロントエンドに返す。** フォームの「Budget」表示に使われる。

```typescript
router.post('/run', async (req, res) => {
  const { location, urgency, budget, overridePrice } = req.body;

  // Step 0: Providerから価格取得
  const { price: fetchedPrice } = await fetch(`${PROVIDER_URL}/weather/price`).then(r => r.json());
  const priceForClaude = overridePrice ?? fetchedPrice;
```

**`overridePrice`** は「予算超過シナリオ」チェックボックス用。`999`が渡されるとClaudeは「高すぎる」と判断してSkipする。

```typescript
  // Step 3: 自律支払い
  const evmSigner = privateKeyToAccount(privateKey);  // 秘密鍵 → ウォレットオブジェクト
  const client = new x402Client();
  client.register('eip155:*', new ExactEvmScheme(evmSigner));
  const fetchWithPayment = wrapFetchWithPayment(fetch, client);

  const premiumResponse = await fetchWithPayment(premiumUrl);
  // ↑ これだけで「402受信 → 支払い → 再リクエスト」が自動で行われる
```

**`wrapFetchWithPayment`** がこのデモの核心。通常の `fetch` をラップして、
`402` を受け取ったときに自動で支払いを実行して再リクエストする `fetch` を返す。

### [Agent] `agent/src/services/walletBalance.ts` — USDC残高取得

```typescript
const publicClient = createPublicClient({
  chain: baseSepolia,
  transport: http(),
});
```

**`createPublicClient`（viem）** はブロックチェーンへの読み取り専用クライアント。
秘密鍵なしでオンチェーンのデータを読める。

```typescript
const raw = await publicClient.readContract({
  address: USDC_ADDRESS,  // Base SepoliaのテストUSDCコントラクト
  abi: erc20Abi,          // ERC20の標準ABI
  functionName: 'balanceOf',
  args: [account.address],
});

return Number(raw) / 1_000_000;  // USDCは小数点6桁
```

**`readContract`** でUSDCコントラクトの `balanceOf` 関数を呼び出す。
返ってくる値は `1000000 = 1.0 USDC` なので `1_000_000` で割る。
