# x402 AI Agent Demo システム解説

---

## 1. このシステムが伝えたいこと

> **「AIが自律的にお金を払う」という世界を体験する**

従来のAPIは「無料か有料か」が固定されていた。  
x402は「HTTPの段階で支払いが発生する」新しい仕組みで、  
AIエージェントが**自分で価値判断して支払う**ことができる。

---

## 2. システム概要

```
ユーザー
  ↓ 地域・緊急度を入力
フロントエンド（React :5173）
  ↓ POST /api/agent/run
Agent Server（Node.js :3002）
  ├─ 1. Provider から価格を取得（GET :3001/weather/price）
  ├─ 2. Provider の無料天気を取得（GET :3001/weather/free）
  ├─ 3. Claude APIに判断を依頼（buy / skip）
  ├─ 4. BUYなら x402 で自律支払い → Provider にプレミアムデータを要求
  └─ 5. プレミアム天気を取得して返却
```

---

## 3. x402とは何か

### 通常のHTTPのフロー
```
クライアント → GET /weather/premium
サーバー     → 200 OK（データ返却）
```

### x402のフロー
```
クライアント → GET /weather/premium
サーバー     → 402 Payment Required（支払い情報を返す）
クライアント → 支払い実行（ブロックチェーン上でUSDC送金）
クライアント → GET /weather/premium（支払い証明付き）
サーバー     → 200 OK（データ返却）
```

**ポイント：** 支払いはHTTPのステータスコード `402` をトリガーに自動で行われる。  
アプリ側が「課金処理」を個別実装する必要がない。

---

## 4. アーキテクチャ

```
【Provider Server :3001】          【Agent Server :3002】
  天気APIを提供・販売                  ユーザー側のAIエージェント
  EVM_ADDRESS（受取）                  WALLET_PRIVATE_KEY（支払）
                              ←──     ANTHROPIC_API_KEY（Claude判断）
  /weather/price → 価格公開          /agent/run → フロントから呼ばれる
  /weather/free  → 無料
  /weather/premium → x402保護
        ↑
    Base Sepolia（テストUSDC）

【Frontend :5173】
  ユーザーの操作画面 → Agent Serverに接続
```

- **Provider**：有料APIを提供する側（天気データ販売者）
- **Agent**：ユーザーの代わりに判断・支払いをするAI
- **Facilitator**：支払いの検証をする第三者（`https://x402.org/facilitator`）

---

## 5. x402の登場人物

| 役割 | 説明 | 今回の担当 |
|------|------|-----------|
| Resource Server | 有料APIを提供する | Provider Server（:3001） |
| Client / Agent | 支払いして情報を取得する | Agent Server（:3002） |
| Facilitator | 支払いの検証をする第三者 | `https://x402.org/facilitator` |

## 6. 技術スタック

### フロントエンド

| 技術 | バージョン | 用途 |
|------|-----------|------|
| React | 18 | UIコンポーネント |
| Vite | 5 | ビルドツール・開発サーバー |
| TypeScript | 5 | 型安全な開発 |

**Viteのプロキシ設定：**  
フロントエンドの `/api/...` へのリクエストを自動的にAgent Server（`:3002`）に転送する。  
これによりフロントエンドはAgent ServerのURLを意識しなくてよい。

---

### バックエンド（Provider / Agent 共通）

| 技術 | バージョン | 用途 |

| Node.js | 18+ | サーバーランタイム |
| Express | 4 | HTTPサーバー |
| TypeScript | 5 | 型安全な開発 |
| dotenv | 16 | 環境変数の管理 |

## 7. セキュリティの考え方

| 項目 | 対策 |
|------|------|
| APIキー（Anthropic） | `agent/.env` のみ（フロントに置かない） |
| 秘密鍵 | `agent/.env` のみ（Gitにコミットしない） |
| 受取アドレス | `provider/.env` のみ |
| `.env` | `.gitignore` で除外済み |
| ウォレット | テストネット専用（mainnetの秘密鍵は絶対に使わない） |
| CORS | Provider は :5173 と :3002 のみ許可、Agent は :5173 のみ許可 |
