# x402 AI Agent Demo システム解説

---

## 1. このシステムが伝えたいこと

> **「AIが自律的にお金を払う」という世界を体験する**

従来のAPIは「無料か有料か」が固定されていた。  
x402は「HTTPの段階で支払いが発生する」新しい仕組みで、  
AIエージェントが**自分で価値判断して支払う**ことができる。

---

## 2. システム概要

```
ユーザー
  ↓ 地域・緊急度を入力
フロントエンド（React :5173）
  ↓ POST /api/agent/run
Agent Server（Node.js :3002）
  ├─ 1. Provider の無料天気を取得（GET :3001/weather/free）
  ├─ 2. Claude APIに判断を依頼（buy / skip）
  ├─ 3. BUYなら x402 で自律支払い → Provider にプレミアムデータを要求
  └─ 4. プレミアム天気を取得して返却
```

---

## 3. x402とは何か

### 通常のHTTPのフロー
```
クライアント → GET /weather/premium
サーバー     → 200 OK（データ返却）
```

### x402のフロー
```
クライアント → GET /weather/premium
サーバー     → 402 Payment Required（支払い情報を返す）
クライアント → 支払い実行（ブロックチェーン上でUSDC送金）
クライアント → GET /weather/premium（支払い証明付き）
サーバー     → 200 OK（データ返却）
```

**ポイント：** 支払いはHTTPのステータスコード `402` をトリガーに自動で行われる。  
アプリ側が「課金処理」を個別実装する必要がない。

---

## 4. アーキテクチャ

```
【Provider Server :3001】          【Agent Server :3002】
  天気APIを提供・販売                  ユーザー側のAIエージェント
  EVM_ADDRESS（受取）                  WALLET_PRIVATE_KEY（支払）
                              ←──     ANTHROPIC_API_KEY（Claude判断）
  /weather/free  → 無料              /agent/run → フロントから呼ばれる
  /weather/premium → x402保護
        ↑
    Base Sepolia（テストUSDC）

【Frontend :5173】
  ユーザーの操作画面 → Agent Serverに接続
```

- **Provider**：有料APIを提供する側（天気データ販売者）
- **Agent**：ユーザーの代わりに判断・支払いをするAI
- **Facilitator**：支払いの検証をする第三者（`https://x402.org/facilitator`）

## 6. コード解説

### 6.1 x402ミドルウェア（`provider/src/index.ts`）

```typescript
app.use(
  paymentMiddleware(
    {
      'GET /weather/premium': {      // 保護するエンドポイント
        accepts: [{
          scheme: 'exact',
          price: '$0.01',            // 価格（0.01 USDC）
          network: 'eip155:84532',   // Base Sepolia
          payTo: EVM_ADDRESS,        // 受取ウォレットアドレス
        }],
      },
    },
    new x402ResourceServer(facilitatorClient)
      .register('eip155:84532', new ExactEvmScheme()),
  ),
);
```

**何をしているか：**  
`/weather/premium` に来たリクエストを監視し、支払い証明がなければ `402` を返す。  
支払い証明があればFacilitatorに検証させ、通過したらルートハンドラに渡す。

---

### 6.2 エージェントのメインロジック（`agent/src/routes/agent.ts`）

```typescript
// Step 1: Provider の無料天気を取得
const freeRes = await fetch(`${PROVIDER_URL}/weather/free?location=...`);

// Step 2: Claudeに判断させる
const decision = await decideWithClaude({ weather, urgency, price, budget });

// Step 3: BUYなら自律支払い
if (decision.buy) {
  const evmSigner = privateKeyToAccount(privateKey);  // 秘密鍵からウォレット生成
  const client = new x402Client();
  client.register('eip155:*', new ExactEvmScheme(evmSigner));
  const fetchWithPayment = wrapFetchWithPayment(fetch, client);  // 402自動処理fetch

  const premiumResponse = await fetchWithPayment(premiumUrl);  // 402→支払→再リクエスト を自動実行
}
```

**ポイント：** `wrapFetchWithPayment` がラップされた `fetch` を返す。  
このfetchは `402` を受け取ったとき、自動で支払いして再リクエストする。

---

### 6.3 Claude APIによる判断（`agent/src/services/aiDecision.ts`）

```typescript
const prompt = `あなたは天気データの購入を判断するAIエージェントです。
以下の状況でプレミアム天気データを購入すべきか判断してください。

現在の情報：
- 無料天気データ：${weather}
- 緊急度：${urgency}（low / medium / high / critical）
- プレミアムデータの価格：${price} USDC
- 予算上限：${budget} USDC

判断基準：
- stormy・tornado などの異常気象は緊急度に関わらず購入する
- rainy は緊急度が high 以上なら購入する（low・medium なら不要）
- cloudy・sunny・clear は緊急度が high 以上でないと購入不要
- 予算を超える支払いは絶対にしない`;

const message = await anthropic.messages.create({
  model: 'claude-haiku-4-5-20251001',
  max_tokens: 200,
  messages: [{ role: 'user', content: prompt }],
});
```

**ポイント：**
- モデルは `claude-haiku-4-5-20251001`（判断用途に最適・低コスト）
- JSON形式で回答させることで、コードから扱いやすくしている
- `buy: true/false` と `reason`（理由）を返させる

### 6.4 天気のモック生成（`provider/src/services/mockWeather.ts`）

```typescript
// 都市名の「文字数 × 先頭文字コード」で天気を決定（再現性あり）
function getCondition(location: string): WeatherCondition {
  const index = (location.length * location.charCodeAt(0)) % conditions.length;
  return conditions[index];
}
```

**なぜこの実装か：**
- 同じ都市名を入力すると必ず同じ天気になる（デモの再現性）
- 外部APIに依存しない（ネット不要・速い）
- 都市名を変えるだけで異なる天気を試せる

```
Tokyo: 5 × 84(T) = 420, 420 % 6 = 0 → 快晴
Paris: 5 × 80(P) = 400, 400 % 6 = 4 → 嵐
```

## 8. データフロー（詳細）

```
[1] ユーザーが入力（location, urgency）

[2] POST /api/agent/run → Agent Server

[3] GET Provider/weather/price
    → 価格を取得（Provider が一元管理）

[4] GET Provider/weather/free?location=...
    → 文字数 × 文字コードで天気を決定（モック）

[5] decideWithClaude({ weather, urgency, price, budget })
    → Claude API に JSON で質問
    → { buy: true/false, reason: "..." } が返る

[6] buy === false の場合
    → そのまま結果を返す（支払いなし）

[7] buy === true の場合
    → wrapFetchWithPayment で Provider/weather/premium を叩く
    → x402ミドルウェアが 402 を返す
    → @x402/fetch が自動で USDC 支払い
    → 支払い証明付きで再リクエスト
    → Facilitator が検証
    → プレミアムデータが返る

[8] フロントエンドに結果・ログを返却
```

## 9. セキュリティの考え方

| 項目 | 対策 |
|------|------|
| APIキー（Anthropic） | `agent/.env` のみ（フロントに置かない） |
| 秘密鍵 | `agent/.env` のみ（Gitにコミットしない） |
| 受取アドレス | `provider/.env` のみ |
| `.env` | `.gitignore` で除外済み |
| ウォレット | テストネット専用（mainnetの秘密鍵は絶対に使わない） |
| CORS | Provider は :5173 と :3002 のみ許可、Agent は :5173 のみ許可 |
